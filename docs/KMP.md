# KMP 算法核心原理剖析

KMP 算法（Knuth-Morris-Pratt）的核心优势在于：**在字符串匹配过程中，主串指针 `i` 永远不会回退（单调递增），从而将传统暴力匹配的 $O(N \times M)$ 复杂度降至 $O(N + M)$。**

算法整体分为两部分：

1. **预处理阶段**：模式串自己和自己匹配，推导出 `next` 数组（时间复杂度 $O(M)$）。
2. **主匹配阶段**：利用 `next` 数组指导模式串滑动，与主串进行匹配（时间复杂度 $O(N)$）。

---

## 1. 核心概念与 `next` 数组求解 (预处理)

计算 `next` 数组的核心本质是**动态规划（DP）**，即利用已经算出的历史状态，推导出下一个状态，绝对不进行多余的重复比较。

### 1.1 代码骨架

```cpp
inline void get_next(String* P, int next[]) {
    if (P == NULL || P->len == 0) return;
    
    next[0] = -1; // -1 作为退无可退的哨兵标记
    int i = 0;
    int j = -1;
    
    while (i < P->len - 1) {
        if (j == -1 || P->ch[i] == P->ch[j]) {
            i++;
            j++;
            next[i] = j; 
        } else {
            j = next[j]; // 失配，查表回退
        }
    }
}
```

### 1.2 核心变量的物理意义

在 `while (i < P->len - 1)` 循环中，始终维持着一个核心不变式：**`j = next[i]`**。

* **`i`**：当前正在计算 `next[i+1]`。可以看作是**后缀**的末尾指针。
* **`j`**：拥有双重身份的游标：
  1. 它是子串 `P[0...i-1]` 的**最长相等前后缀的长度**。
  2. 它刚好也是前缀的**下一个待匹配字符的索引**。

### 1.3 状态转移方程解读

当我们要计算包含新字符 `P[i]` 的最长相等前后缀（即 `next[i+1]`）时，我们需要比较 `P[i]`（新后缀的末尾）和 `P[j]`（前缀的下一个字符）。

* **情况 A：`P[i] == P[j]` (顺利延展)**
  * **原理**：新加入的字符正好匹配，原有的最长相等前后缀在两端同时增加了一个字符。
  * **转移**：`i++; j++; next[i] = j;` （长度加 1，并记录）。
* **情况 B：`P[i] != P[j]` (失配与回退 —— 灵魂所在！)**
  * **面临问题**：长度为 `j` 的前缀无法继续延展，我们需要找一个**次长**的相等前后缀来试试。
  * **为什么是 `j = next[j]`？**
    * 在失配发生前，我们已知 `P[0...j-1] == P[i-j...i-1]`（前缀等于后缀）。
    * 既然完整的长度 `j` 行不通，寻找一个更短的匹配前缀 $L$ ($L < j$) 使其能和当前后缀匹配，**在数学上等价于：在原本的那个前缀 `P[0...j-1]` 内部，寻找它自己的最长相等前后缀！**
    * 而“`P[0...j-1]` 的最长相等前后缀长度”这个值，早就被我们算过了，它就存在 **`next[j]`** 里！
  * **转移**：`j = next[j];` （瞬间回退到次长前后缀的长度，在下一轮循环继续尝试匹配，直到 `j` 退无可退变为 `-1`）。

## 2. KMP 的基本框架 (主匹配过程)

KMP 算法的主匹配阶段，是一个典型的时间复杂度为 $O(N)$ 的状态机流转过程。
当发生失配时，主串指针 `i` 原地不动，只有模式串指针 `j` 会根据 `next` 数组的指导进行滑动。

### 2.1 核心代码骨架

```cpp
inline int KMP(String* S, String* P) {
    if (S == NULL || P == NULL || S->len == 0 || P->len == 0) return -1;
    
    int next[P->len];     // C99 VLA: 在栈上高效分配 next 数组
    get_next(P, next);    // 预处理模式串，生成 next 数组

    int i = 0; // 主串 S 的游标（永远不回退）
    int j = 0; // 模式串 P 的游标（代表当前已匹配的字符数）

    while (i < S->len && j < P->len) {
        if (j == -1 || S->ch[i] == P->ch[j]) {
            i++;
            j++;
        } else {
            j = next[j]; // 发生失配，查表滑动
        }
    }

    if (j == P->len) {
        return i - j; // 匹配成功，返回在主串中的起始索引 (0-based)
    }
    return -1; // 匹配失败
}
```

### 2.2 核心变量与指针流转解析

在 `while` 循环中，两个指针 `i` 和 `j` 的流转逻辑分为两个核心分支：

* **分支 A：携手前进 (`S->ch[i] == P->ch[j]`) 或 重新开始 (`j == -1`)**
  * **当 `S[i] == P[j]` 时**：当前字符匹配成���。主串和模式串的游标双双向后移动一步，准备比较下一个字符。
  * **当 `j == -1` 时（极度优雅的哨兵设计）**：
    * 当主串的 `S[i]` 与模式串的**第一个字符** `P[0]` 就发生失配时，执行分支 B 会让 `j = next[0]`，也就是 `j = -1`。
    * 此时进入分支 A，执行 `i++` 和 `j++`。主串指针向前走了一步 (`i = i + 1`)，模式串指针回到了起点 (`j = -1 + 1 = 0`)。
    * **意义**：完美实现了“放弃当前主串字符，从下一个主串字符重新开始匹配”的逻辑，避免了冗余的特判语句。

* **分支 B：发生失配，查表滑动 (`S->ch[i] != P->ch[j]`)**
  * **动作**：主串指针 `i` **原地站住不动**。模式串指针 `j` 直接通过查表 `j = next[j]` 瞬间滑到一个最优的历史位置。
  * **意义**：跳过那些根据模式串自身对称性“注定会失败”的重复比较，直接让模式串的前缀对齐主串刚刚匹配过的后缀。

### 2.3 匹配成功的返回值计算 (`i - j`)

```cpp
if (j == P->len) {
    return i - j; 
}
```

* 当 `j` 达到了模式串的长度 `P->len` 时，说明模式串的所有字符都匹配完了。
* **为什么起始索引是 `i - j`？**
  * 在最后一次成功匹配（即匹配模式串最后一个字符）后，代码依然无情地执行了一次 `i++` 和 `j++`。
  * 导致退出循环时，主串指针 `i` 已经越过了最后一个匹配字符，停在了下一个位置；而此时 `j` 刚好等于模式串的总长度。
  * 因此，当前的 `i` 往回倒退一整个模式串的长度（减去 `j`），就精准落在了首个匹配字符的起始下标位置（基于 0 的索引）。
