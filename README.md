# learnAlgorithm

这是一个专注于基础数据结构与算法实现的练习项目，参考《王道2027数据结构》。
相关算法的实现采用纯c风格，极少使用c++特性，但为了方便测试和代码管理，将项目组织成c++项目的形式。

## 项目特点

- **代码组织**: 采用 Header-only 风格。所有数据结构与算法的实现均位于 `include/` 目录下的 `.hpp` 文件中，方便集成与调试。
- **实现风格**: 核心逻辑遵循纯 C 的设计哲学，仅少量使用 C++ 引用 (Reference) 与函数重载等特性以提升代码的可读性与工程便利性。
- **质量保障**: 项目采用测试驱动开发 (TDD) 模式，利用 Catch2 框架对每个组件进行详尽的单元测试。
- **工具链**: 使用 CMake 构建系统，并结合 `just` 任务运行器简化日常的开发、编译与测试流程。

## 目录结构

- `include/`: 核心头文件实现，包含链表、栈、队列及调度场算法。
- `tests/`: 基于 Catch2 的单元测试代码。
- `examples/`: 各组件的简单应用示例。
- `third_party/`: 项目依赖库，如 Catch2。

## 编译与运行

项目使用 `just` 自动化常用命令。在运行之前，请确保已安装 `cmake` 和 `just`。

### 1. 初始化配置
首次使用需生成构建目录：
```bash
just setup

```

### 2. 编译

编译所有示例项目与测试用例：

```bash
just build

```

### 3. 运行测试

执行所有自动化测试：

```bash
just test

```

如果需要编译并运行特定的测试目标（例如双向链表测试）：

```bash
just p test_DLinkList

```

### 4. 交互式操作

如果系统中安装了 `fzf`，可以使用以下指令进行交互式选择：

* `just run`: 交互式选择并运行可执行文件。
* `just build-p`: 交互式选择并编译目标。

### 5. 清理

删除所有构建产物：

```bash
just clean
```
